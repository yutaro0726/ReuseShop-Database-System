# デプロイ手順・運用ガイド

本番環境へのデプロイ・運用に関する詳細な手順と注意点をまとめます。

---

## 本番環境の要件

- PHP 7.4以上（mbstring, PDO拡張必須）
- MySQL 5.7以上
- Apache（.htaccess対応必須）または同等のWebサーバー
- ファイル書き込み権限（storage/ 配下）
- 外部パッケージ管理ツール（Composer等）は不要

---

## デプロイ手順

1. 最新ソースコードをサーバーにアップロード
2. `db/schema.sql`を用いてデータベースを作成・初期化
3. `config/env.php`を本番用に編集（APP_ENV, DB接続情報等）
4. `public/`をWebサーバーのドキュメントルートに設定
5. `storage/`ディレクトリに書き込み権限を付与
6. `.htaccess`の内容を確認し、不要なディレクトリへのアクセスを遮断
7. 動作確認（トップページ、ログイン、レビュー投稿等）

---

## 設定変更点

- `config/env.php`  
  - APP_ENVを`production`に設定
  - 本番用DB接続情報を記載
- `config/database.php`  
  - 必要に応じてDB設定を調整
- `.htaccess`  
  - セキュリティ・ルーティング設定を本番向けに最適化

---

## 確認項目

- [ ] APP_ENVがproductionになっている
- [ ] DB接続情報が正しい
- [ ] public/以外にWebからアクセスできない
- [ ] storage/に書き込み権限がある
- [ ] エラーメッセージが外部に表示されない
- [ ] 不要なファイルが残っていない
- [ ] セキュリティ対策（[docs/security.md](security.md)）が有効

---

## 環境依存の注意点

- Windows環境（XAMPP等）とLinux環境でパスや権限設定が異なる場合あり
- 本番環境でのパーミッション設定・ファイル所有者に注意
- .htaccessが無効な場合はWebサーバー設定でアクセス制御を行うこと
