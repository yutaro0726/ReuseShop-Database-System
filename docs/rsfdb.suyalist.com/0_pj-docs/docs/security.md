# セキュリティ対策

本アプリケーションで実施している主なセキュリティ対策と、運用時のチェックリスト・注意点をまとめます。

---

## 実装済みセキュリティ対策一覧

- **CSRF対策**  
  フォーム送信時にトークンを付与し、サーバー側で検証
- **XSS対策**  
  出力時にHTMLエスケープを徹底
- **SQLインジェクション防止**  
  プリペアドステートメント利用
- **パスワード管理**  
  パスワードはハッシュ化して保存（平文保存禁止）
- **セッション管理**  
  セッション固定攻撃対策、セッションID再生成
- **ファイルアップロード制限**  
  拡張子・MIMEタイプ・ファイルサイズ検証、保存先制限
- **管理画面認証**  
  管理者権限の認証・認可
- **エラーメッセージ制御**  
  本番環境では詳細なエラーを非表示
- **.htaccessによるアクセス制御**  
  public/配下以外への直接アクセス禁止

---

## 運用チェックリスト

- [ ] APP_ENVがproductionになっているか
- [ ] config/env.phpのDB情報・キーが漏洩していないか
- [ ] public/以外にWebからアクセスできないか
- [ ] デバッグ情報・エラーログが外部に公開されていないか
- [ ] 不要なファイル（テスト・サンプル等）が残っていないか
- [ ] アップロードファイルの内容・拡張子チェックが有効か
- [ ] 管理者パスワードが十分に強力か

---

## 注意点・推奨事項

- 本番環境では必ずAPP_ENV=productionに設定
- DB接続情報や各種キーは外部に公開しない
- 定期的にパスワード・権限を見直す
- サーバー・ミドルウェアのセキュリティアップデートを適用
- 外部サービス連携時はAPIキー等の管理に注意
- バックアップ・障害時の復旧手順を整備
