## ディレクトリ別詳細説明

### 1. 開発環境専用ディレクトリ

#### `docs/` - ドキュメント管理
- **用途**: プロジェクト全体のドキュメント管理
- **公開**: 本番環境では非公開
- **サブディレクトリ**:
  - `work-report/`: 作業記録・進捗管理
  - `References/`: 参考文献・関連資料
  - `sample/`: 外部サイトサンプル
  - `design/`: 設計資料（ER図、相関図等）

#### `mysql/` - データベース関連
- **用途**: SQLファイル、データベース設計書の管理
- **公開**: 本番環境では非公開
- **ファイル形式**: .sql, .md

### 2. 非公開ディレクトリ

#### `system/` - システム設定・ライブラリ
- **用途**: 機密情報、設定ファイル、ライブラリの配置
- **公開**: Webからアクセス不可
- **サブディレクトリ**:
  - `config/`: データベース接続情報等
  - `lib/`: 外部ライブラリ
  - `classes/`: システム用クラスファイル

### 3. 公開ディレクトリ

#### `revi.mypressonline.com/` - Webルート
- **用途**: 一般ユーザーがアクセス可能なファイル
- **公開**: Web公開ディレクトリ
- **主要ファイル・ディレクトリ**:
  - `index.php`: サイトトップページ
  - `admin/`: 管理画面
  - `recycle-shop/`: リサイクルショップ関連ページ
  - `assets/`: CSS、JavaScript、画像等
  - `api/`: API エンドポイント

## 重要な運用ルール

### セキュリティ
- 機密情報は必ず `system/` ディレクトリに配置
- データベース接続情報は設定ファイルで一元管理
- パスワード等はハードコーディング禁止

### ファイル管理
- UTF-8 (BOM無し) でファイル作成
- ファイル名は小文字とアンダースコア、ハイフンのみ
- 使用しないファイルは `old/` フォルダに移動または削除

### データベース
- 文字コードは `utf8mb4_general_ci` (絵文字対応)
- SQLインジェクション対策でプリペアドステートメント使用
- テーブル構造変更時は必ずドキュメント更新
