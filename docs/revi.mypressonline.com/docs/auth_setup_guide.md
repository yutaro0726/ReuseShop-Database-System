## 統合認証システムの概要

### 認証フロー
1. **Basic認証チェック**: 管理画面アクセス時に最初に確認
2. **ユーザーセッション確認**: ログイン状態の検証
3. **権限レベル判定**: アクセス可能な機能の確認
4. **アクセス制御実行**: 適切な画面への誘導

### 権限レベル構成
```
super_admin (100)    - システム全体管理者
total_admin (90)     - 総合レビューサイト管理者
recycle_admin (80)   - リサイクルショップ管理者
editor (50)          - レビュー編集者
viewer (10)          - 閲覧のみ
```

## 設定ファイル構成

### 主要設定ファイル
- `system/config/auth_config.php` - 統合認証設定
- `system/config/admin_config.php` - 管理画面設定（既存）
- `revi.mypressonline.com/admin/.htaccess` - Basic認証設定（既存）

### 新機能設定
- **SSOライク認証**: ユーザーログイン済みの場合、Basic認証をスキップ可能
- **セッション管理**: 1時間（3600秒）のセッション有効期限
- **失敗時ロックアウト**: 5回失敗で15分間のアクセス制限

## セキュリティ設定

### 現在の設定（開発段階）
- パスワード最低文字数: 8文字
- 特殊文字必須: 無効（開発段階のため）
- IPホワイトリスト: 無効（全IP許可）
- CSRF保護: 有効

### 本番環境での推奨設定
```php
'password_require_special' => true,  // 特殊文字必須
'ip_whitelist' => ['xxx.xxx.xxx.xxx'], // 許可IPのみ
'cookie_secure' => true,  // HTTPS必須
```

## ログ設定

### ログファイル場所
- 認証ログ: `system/logs/auth.log`
- 管理ログ: `system/logs/admin.log`

### ログ内容
- ログイン成功/失敗
- Basic認証試行
- 権限不足アクセス
- 管理操作履歴

## 運用手順

### 初回セットアップ
1. `system/config/auth_config.php`の設定確認
2. ログディレクトリの権限設定
3. Basic認証情報の配布（機密情報として）
4. 動作テストの実施

### 定期メンテナンス
- 週次: ログファイルの確認とローテーション
- 月次: パスワード変更の検討
- 随時: 不正アクセスログの監視

### トラブルシューティング

#### Basic認証が効かない場合
1. `.htaccess`ファイルの確認
2. サーバーのmod_auth設定確認
3. ログファイルでエラー内容確認

#### ログインできない場合
1. パスワードの再確認（大文字小文字区別）
2. セッションの清理（ブラウザのクッキー削除）
3. ログファイルで詳細エラー確認
